# plan

Active planning and tracking documents for the Goal Tracker project. This folder contains only forward-looking content: open requirements, outstanding bugs, and backlogged features.

| File / Folder | Purpose |
|---|---|
| [`BACKLOG.md`](BACKLOG.md) | Future features and enhancements not in the current release, organised by phase and priority. Add new ideas here. |
| [`BUG_TRACKER.md`](BUG_TRACKER.md) | Open bugs awaiting a fix. Bugs are removed once a test is created to verify the resolution. |
| [`PROJECT_BRIEF.md`](PROJECT_BRIEF.md) | Full project requirements: core features (goal management, progress tracking, retroactive logging, reversion, reporting), tech stack decisions, and MVP scope. |
| [`ROADMAP.md`](ROADMAP.md) | High-level development roadmap listing unprepared, prepared, and planning-ready requirements. |

## Relationships

For comprehensive developer documentation (architecture, services, components, deployment) see [`../docs/`](../docs/README.md). Bug fixes should also be reflected in `BUG_TRACKER.md` once resolved.
